## 遇到的问题 & 解决方案

其实大部分使用过程中出的问题都可以直接让他看日志进行分析。

### 网关的访问

因为对现在不完善 AI 的不信任，我也没有在我的台式电脑上装，而是在服务器上。
然后我发现最简单的方法还是用 SSH 做端口转发：

```bash
ssh -L 18789:127.0.0.1:18789 user@ip
```

不然的话还需要把 http 的问题搞定（默认不给直接访问 http，因为这样通信是裸的，不安全）

### 权限不够

第二次装上 OpenClaw 后，发现 exec 执行不了。
然后看到 3 月 2 号更新的版本里，默认只给 tools 比较低的权限，需要把 tools.profile 改成 full。

### 上传文件失败

飞书插件里是有做白名单的，除了 workspace 目录外，其他目录，包括 `/tmp` 下的内容都是无法被发送出来的，目前暂时没有相关的解决方法，因为代码层面压根也没有给配置白名单的地方，只能让 agent 把文件复制到 workspace 再发给我了。

### 无法接收文件

看日志会看到报 411 错误码，让 openclaw 自己 debug，最后发现是 systemctl 里启动加了一行 https_proxy 导致的，去掉再重启就好了。
